# Персональный Трекер на Django

Веб-приложение для управления привычками, финансами и недельным расписанием с уникальной системой мотивации "Матрёшка".

## Структура проекта

```
tracker_project/
├── manage.py              # Django management script
├── requirements.txt       # Зависимости Python
├── db.sqlite3            # База данных SQLite (создаётся после миграций)
├── tracker_api/          # Основное приложение
│   ├── models.py         # Модели данных (Habit, Transaction, WeekTask, WeekArchive)
│   ├── services.py       # Сервисный слой с бизнес-логикой
│   ├── views.py          # API endpoints
│   └── urls.py           # URL маршруты API
├── tracker_project/      # Конфигурация проекта
│   ├── settings.py       # Настройки Django
│   ├── urls.py           # Корневые URL
│   └── wsgi.py           # WSGI конфигурация
├── templates/tracker/    # HTML шаблоны
│   └── index.html        # Основной шаблон приложения
└── static/               # Статические файлы
```

## Установка и запуск

### 1. Создание виртуального окружения

```bash
cd tracker_project
python -m venv venv
```

### 2. Активация виртуального окружения

**Windows:**
```bash
venv\Scripts\activate
```

**macOS/Linux:**
```bash
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Применение миграций

```bash
python manage.py makemigrations
python manage.py migrate
```

### 5. Запуск сервера разработки

```bash
python manage.py runserver
```

Приложение будет доступно по адресу: http://127.0.0.1:8000/

## Функционал

### 🔹 Трекер привычек
- Добавление и удаление привычек
- Отметка выполнения на каждый день
- Подсчёт серии дней (streak)
- Статистика выполнения

### 📅 Недельное расписание
- Просмотр задач на всю неделю
- Добавление задач с указанием времени
- Навигация между неделями
- Быстрый переход к любой дате
- Автоматический архив завершённых недель
- Прогресс выполнения дня

### 💰 Бюджет
- Учёт доходов и расходов
- Подсчёт баланса
- История транзакций

### 🪆 Система "Матрёшка"
- Динамическое изменение настроения в зависимости от прогресса
- 4 уровня: от спокойного до бешенства
- Мотивирующие (и не очень) сообщения

## Архитектура

Проект построен с соблюдением принципов ООП:

- **Наследование**: базовая модель `BaseModel` для общих полей
- **Инкапсуляция**: сервисный слой (`services.py`) скрывает бизнес-логику
- **Полиморфизм**: единый интерфейс API для разных типов данных
- **DRY**: переиспользование кода через сервисы и утилиты

## API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/habits/` | Получить все привычки |
| POST | `/api/habits/` | Создать привычку |
| POST | `/api/habits/<id>/` | Переключить статус привычки |
| DELETE | `/api/habits/<id>/` | Удалить привычку |
| GET | `/api/transactions/` | Получить транзакции |
| POST | `/api/transactions/` | Создать транзакцию |
| DELETE | `/api/transactions/<id>/` | Удалить транзакцию |
| GET | `/api/week-schedule/` | Получить расписание недели |
| POST | `/api/week-tasks/create/` | Создать задачу |
| POST | `/api/week-tasks/<id>/toggle/` | Переключить задачу |
| DELETE | `/api/week-tasks/<id>/delete/` | Удалить задачу |
| GET | `/api/archive/` | Получить архив недель |
| POST | `/api/archive/clear/` | Очистить архив |
| GET | `/api/matryoshka/` | Получить состояние Матрёшки |

## Технологии

- **Backend**: Django 4.2+
- **Database**: SQLite (по умолчанию)
- **Frontend**: Vanilla JavaScript, CSS3
- **API**: REST-like JSON API
